# 📌 프로젝트 소개
<br>

## **프로젝트**

**전기차 & 충전소 현황 분석**<br>
전기차와 충전소 간의 상관관계 및 인프라 분석

---

## **기간**

2025년 7월 3일(목) ~ 2025년 7월 4일(금)<br>

---

## **목표**

1. 전기차 관련 대중의 관심사 및 주요 이슈 파악<br>
2. 전기차 충전 인프라 현황 및 변화 추이 파악<br>
3. 전기차 보급 및 충전 인프라 간의 상관관계 분석<br>
4. 전기차와 충전소 데이터 분석 대시보드 제작<br>

# 🖊 회고
<br>

| 이름 | 회고 |
|---|---|
| 문승현 | 이번 프로젝트는 혼자 하던 공부와 달리, 목적을 가지고 처음으로 진행한 팀 프로젝트라는 점에서 의미가 컸다. 처음엔 협업이 많이 어려울 줄 알았지만, 생각보다 원활하게 소통하며 맡은 역할에 책임감을 갖고 임했다. 다만, 일부 아쉬운 부분도 분명히 있었고 더 잘할 수 있었던 점들이 눈에 보였다. 다음 프로젝트에서는 이런 부분들을 보완하고 싶다. 나는 전기차와 충전기에 관련한 여러 데이터들을 기반으로 DB를 구축했고 또한 ERD 설계, 파이썬 코드 구현을 맡았다. 이론에서 배운 내용을 직접 기능을 구현해보며 실제로 적용해보는 소중한 경험을 할 수 있었다. |
| 신희석 | 이번 프로젝트를 하게 되었을 때 다른 팀원들에게 방해가 될 것 같다는 두려움이 있었습니다. 키워드를 크롤링하는 것부터 히트맵을 만드는 역할을 할 때, 강의 때 배운 것들을 토대로 조금 더 응용해 봄으로 조금 더 성장할 수 있었던 것 같습니다. 스스로 공부해가며 결과물을 만들었을 때 뿌듯했습니다. 이론으로 실습으로 따라하던 것들을 스스로 처음부터 했을 때, 그리고 화면에 나타날 때 희열을 느꼈으며 더 많은 것들을 해보고 싶다는 생각이 들었습니다. 앞으로 오는 프로젝트에서는 조금 더 많이 성장해 프로젝트에 많은 기여를 하고 싶다는 생각이 들었습니다. |
| 안주영 | 팀장으로서 가장 주력한 부분은 의견 조율이었습니다. 개인적으로는 목표치 이상의 성과를 내고 싶었지만, 기한이라는 현실 사이에서 이상과 현실의 간극을 좁혀야 했습니다. 이 과정에서 저는 중요한 프로젝트 문화를 배웠습니다. 우선적으로 핵심 요구사항을 달성하고, 남은 시간에 프로젝트를 고도화하는 방식이었습니다. 덕분에 짧은 기간에도 팀원 각자의 역량을 뛰어넘는 결과물을 얻을 수 있었고, PM으로서 한 단계 성장했다고 느낍니다. 또한, 코드 통합 및 디버깅을 통해 문제 해결 능력을 향상시킬 수 있었습니다. 이는 프로젝트 전반에 대한 이해를 높여 팀원들의 어려움을 해결하는 데 기여했습니다. 팀원들의 적극적인 참여와 열정적인 토론 덕분에 좋은 시너지를 낼 수 있었고, 그런 팀원들에게 진심으로 감사합니다. |
| 이현민 | 이번 프로젝트에서 데이터 시각화와 Streamlit 기반 웹 애플리케이션 개발을 담당하며 다양한 기술 스택을 경험할 수 있었습니다. KoNLPy를 활용한 한글 형태소 분석을 통해 WordCloud로 텍스트 시각화를 구현했고, Pandas를 활용한 데이터 전처리 및 Folium을 이용한 지도 구현을 통해 데이터 분석 전 과정을 겪을 수 있었습니다. 개발 과정에서 한글 텍스트 인코딩과 같은 난관에 부딪히기도 했지만, 이를 해결하는 과정에서 문제 해결 역량을 강화할 수 있었고, UI를 담당하며 기능 구현만큼이나 사용자 중심의 인터페이스 설계가 중요하다는 것을 깨달으며 전체적인 그림을 그리는 시야를 넓힐 수 있었습니다. |
| 임종민 | 짧은 일정이었지만, 프로젝트를 통해 지금까지 배운 내용들을 구체적으로 실무에 어떻게 활용할 수 있는지 알 수 있는 좋은 기회였습니다. 코드를 작성하는 데 AI의 도움을 받아 높은 생산성을 유지할 수 있었고, 조원들 모두의 집중력과 헌신 덕분에 무사히 프로젝트를 완수할 수 있었습니다. 특히, 데이터를 확보하는 과정과 이를 분석과 시각화하는 데 모두가 세심하게 고민해준 덕분에 좋은 결과를 낼 수 있었고, 일련의 과정을 통해 사용자 관점에서 생각하는 계기도 됐습니다. 함께해준 조원들에게 진심으로 감사드립니다. |
